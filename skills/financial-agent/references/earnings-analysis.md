# Earnings analysis

1. Fix the reporting period and cutoff time.
2. Compare actuals with pre-result expectations metric by metric; cite the source and unit for both.
3. Separate reported facts from management framing and analyst interpretation.
4. Compare prior guidance, new guidance, and the assumptions embedded in each. “Maintained” is a real outcome.
5. Judge beat quality: recurring versus one-off, revenue versus margin, cash conversion, dilution, working capital, and guidance pull-forward.
6. Explain what changes the thesis and what is only price movement or quarterly noise.
7. List risks, invalidation conditions, and unknowns. Name the next data point.
8. Populate templates/earnings-report.md and an earningsReport JSON payload. Run npm run reports:validate -- --type earningsReport --input <file>.
